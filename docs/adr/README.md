- [001](001-csharp-codegen-in-consumer.md) — Moved ClangSharp from Gradle to .targets in NuGet (later replaced by KSP)

- [004](004-cir-intermediate-representation.md) — Type-safe C# AST (aligned with Swift Export's SIR)
- [005](005-object-return-semantics.md) — New wrapper per access; identity not preserved (matches ObjC/Swift)
- [006](006-enum-mapping.md) — Ordinal-backed C# enum + extension methods for properties
- [007](007-top-level-function-class-naming.md) — File name as class name; `Kt` suffix only on conflict
- [008](008-data-class-mapping.md) — Regular class + delegate to Kotlin; explicit `Copy()` instead of C# record
- [009](009-sealed-class-mapping.md) — Abstract base + nested sealed subclasses with runtime discriminator
- [010](010-generics-mapping.md) — Type-erased bridge + generic C# class
- [011](011-collection-type-mapping.md) — Eager copy with interface return types mirroring Kotlin mutability
- [012](012-lambda-function-type-mapping.md) — Opaque handle for Kotlin→C#; function pointer deferred to Phase 5
- [013](013-extension-property-mapping.md) — `GetXxx()`/`SetXxx()` extension methods for Kotlin extension properties
- [014](014-value-class-mapping.md) — `readonly record struct` with unwrapped bridge for Kotlin value classes
- [015](015-generic-type-constraint-mapping.md) — C# `where` clauses from Kotlin upper bounds; `CirTypeParameter` model
- [016](016-generic-variance-mapping.md) — `out`/`in` variance on C# interfaces only; dropped on classes (C# restriction)
- [017](017-inline-function-mapping.md) — Inline functions treated as regular functions; existing pipeline handles them
- [018](018-type-alias-mapping.md) — Transparent expansion to underlying type; aliases erased (matches all interop targets)
- [019](019-suspend-function-mapping.md) — Callback-based bridge with TaskCompletionSource; suspend → Task/async; coroutine launched on Dispatchers.Default
- [020](020-suspend-lambda-mapping.md) — `KotlinSuspendFunc<R>` / `KotlinSuspendAction` with `InvokeAsync()` for suspend lambda properties
- [021](021-structured-concurrency.md) — Scope-per-class with `SupervisorJob`; `Dispose()` cancels in-flight coroutines; `ObjectDisposedException` guard
- [022](022-cancellation-token-support.md) — `CancellationToken` parameter on async methods; Job StableRef returned from export; `nuget_job_cancel` shared export
- [023](023-exception-propagation.md) — `KotlinException.KotlinType` property; `Pair<String,String>` error bridge; `nuget_error_type`/`nuget_error_message` exports
- [024](024-sync-exception-propagation.md) — Error out-parameter for sync functions; wrap-all (no `@Throws` required); mirrors ObjC `NSError**` ABI
- [025](025-async-disposable.md) — `IAsyncDisposable` + `DisposeAsync()` for graceful drain; `nuget_scope_drain` export joins all child jobs before releasing handles
- [026](026-flow-mapping.md) — `Flow<T>` → `KotlinFlow<T>` (`IAsyncEnumerable<T>`); Channel-buffered push→pull bridge; three-callback ABI (`onNext`, `onComplete`, `onError`)
- [027](027-stacktrace-propagation.md) — `KotlinException.KotlinStackTrace` property; `Triple<String,String,String>` error bridge; `nuget_error_stacktrace` export; `ToString()` override appends Kotlin trace
- [028](028-exception-cause-chain.md) — `e.cause` → `KotlinException.InnerException`; recursive `NugetError` data class error bridge (replaces `Triple`); `nuget_error_cause_*` exports; each cause a full `KotlinException`
- [029](029-exception-type-mapping.md) — `IKotlinException` interface; 8 Kotlin stdlib exceptions mapped to .NET analogs (e.g. `IllegalArgumentException` → `KotlinArgumentException : ArgumentException`); `KotlinException` fallback for unmapped types; `BuildMapped` switch in generated helper
- [030](030-property-exception-propagation.md) — ADR-024 error-out-parameter extended to property getter/setter exports; both callable forms of nullable two-call pattern wrapped independently
- [031](031-constructor-exception-propagation.md) — Constructor export returns `COpaquePointer?` (null on error) + `errorOut`; C# constructor checks error before assigning `_handle`; data class `copy()` receives same treatment
- [032](032-generic-constructor-exception-propagation.md) — ADR-031 error-out applied to all 14 generic `create_*`/`create_object` exports; C# error check centralized in `NugetMarshal.CreateBox<T>` (unconstrained) + per-constructor for constrained `Create_object`
- [033](033-value-class-constructor-exception-propagation.md) — Value-class ctor exceptions via private `CreateChecked` helper called from `: this(...)` (keeps `new Email(...)`, no static-factory redesign); primitive underlying uses ADR-024 dummy-return pattern
- [034](034-secondary-constructor-exceptions.md) — Export all constructors via `getConstructors()`; primary keeps `_create`, secondaries get `_create_N`; ADR-031 error-out per constructor; fail-fast KSP error on C# signature collision
- [035](035-value-class-primary-constructor-validation.md) — Value-class primary `init` enforced by switching to a hand-written `record struct` where every ctor has a validating body (also removes ADR-033's `: this(...)` delegation); get-only property prevents `with`/initializer bypass; reference-underlying deferred
- [036](036-reverse-interop-mechanism.md) — C#→Kotlin reverse interop via delegate + `GCHandle.Alloc` + `Marshal.GetFunctionPointerForDelegate`; consistent with existing Flow/async callback pattern; `[UnmanagedCallersOnly]` and vtable interface bridging deferred
- [037](037-stored-callbacks.md) — Stored callbacks (event handlers/observers) via `IDisposable` subscription handle; `add*`/`remove*` pair detection; bridge-lambda registry on Kotlin side; no new Kotlin→C# mechanism needed
- [038](038-aot-compilation.md) — NativeAOT compatibility; generics reflection path (`GetField`/`Activator`) is the concrete trimmer blocker; `CSharpProfile` makes the dialect (`DllImport`→`LibraryImport`, generic dispatch) a generation-time choice; deferred
- [039](039-interface-bridging.md) — C# implementing Kotlin interfaces via N flat function pointers per method; `add*/remove*` pair detection (same as ADR-037); Kotlin bridge generates anonymous `object : Interface`; same `IListener` for both directions; `IDisposable` subscription
- [040](040-interface-return-type-mapping.md) — Kotlin returning interface-typed values to C#; concrete `sealed class Foo : IFoo` per interface; Kotlin interface-dispatch exports (`asStableRef<Foo>().get()`); `IPet?` return type; fixes current broken `new Pet(handle)` compile error
- [041](041-kotlin-to-csharp-call-mechanism.md) — Phase 8 Kotlin → managed C# calls via `[UnmanagedCallersOnly]` thunks registered at startup from `[ModuleInitializer]`; Kotlin stubs `requireNotNull`-guard the pointer table and fail fast if unregistered; standalone Kotlin hosts requiring `hostfxr` CLR hosting explicitly out of scope for v1
- [042](042-assembly-metadata-extraction.md) — Phase 8 metadata extraction: `dotnet restore` (synthetic `.csproj` → `project.assets.json`) for resolution + bundled dotnet tool using `System.Reflection.Metadata.MetadataReader` to read the ECMA-335 public API without loading the assembly; pure-JVM both halves deferred as Future Improvement
- [043](043-bridgeable-subset-boundary.md) — Phase 8 C-ABI-bridgeable subset: overload sets, `ref struct`/`Span<T>`, open generics, `dynamic`, default interface methods structurally excluded; v1 auto-mappable subset only + per-member fail-fast diagnostics naming each skipped member; Kotlin subclassing a C# class out of scope, implementing a C# interface in scope
- [044](044-nuget-dependency-dsl.md) — Phase 8 NuGet dependency DSL: `dependency()` + `bind {}` opt-in inside `nuget { dependencies { } }`; absent `bind {}` = resolve only; `packageName`/`include`/`exclude`/`alias` in `bind`; downstream data contract for `nugetGen`/`nugetExtractApi` tasks
- [045](045-nuget-resolution-pipeline.md) — Phase 8 resolution pipeline: two-task split `nugetGen` (pure csproj generation, cacheable) + `nugetRestore` (`dotnet restore`, `@DisableCachingByDefault`); `net8.0` TFM floor; conditional `<RestoreSources>`; `obj/project.assets.json` as inter-task manifest; `nugetImport` umbrella task; config-cache via `ExecOperations`
- [046](046-reverse-ir-model-and-json-contract.md) — Phase 8 Reverse IR: `RirFile`/`RirAssembly`/`RirNamespace`/`RirType`/`RirMethod`/`RirProperty`/`RirTypeRef` sealed data classes in `nuget/` module + `kotlinx.serialization` JSON with `"kind"` discriminator; `parseReverseIr()` + `deriveDllPaths()` pure functions; `NugetExtractApiTask` inputs/outputs + plugin wiring; `nuget-metadata-reader/` C# `MetadataReader` project; five-slice TDD breakdown
- [047](047-per-package-namespace-filters-at-reader-cli.md) — Phase 8 per-package namespace filters: inline stateful `--include`/`--exclude` flags adjacent to each `--package` triple (vs JSON manifest); fixes flattening bug where Package A's filters were applied to Package B's assemblies; `metadataReaderCommand` signature changes from `List<String>` to `Map<String, List<String>>` for includes/excludes; `CliArgs.Parse` in Program.cs becomes stateful
- [048](048-kotlin-stub-generation-from-reverse-ir.md) — Phase 8 Kotlin stub generation: `nugetGenerateBindings` task; `object`-per-type stubs for C# static classes; two-file split (`{TypeName}Bindings.kt` + `{TypeName}.kt`); generated sources into `build/nuget-interop/kotlin/nativeMain/` wired into `nativeMain` source set; `expect`/`actual` `freeManagedString` helper for string deallocation; registration export naming convention `nuget_{ns_snake}_{type_snake}_register`; v1 type-mapping table (RirTypeRef → Kotlin type)
- [049](049-csharp-registration-shim-generation.md) — Phase 8 C#-side registration shim generation: new sibling `nugetGenerateShims` task emitting `{TypeName}Registration.cs` per bound `RirClass`; corrected inverse type-mapping table (`IntPtr` for strings, `byte`/`ushort` narrowing for non-blittable `bool`/`char`); `[UnmanagedCallersOnly]` thunks + `[ModuleInitializer]` calling the Kotlin registration export via `sharedLib.baseName`-targeted `[DllImport]`; catch-and-sentinel exception handling per ADR-041; shim delivery merged into the existing packed-nupkg `contentFiles/cs/any/` convention from ADR-001 (packaging wiring deferred to the "Phase goal" end-to-end ROADMAP item); shared `bridgeableStaticMethods()` extraction to prevent Kotlin/C# parameter-order drift
