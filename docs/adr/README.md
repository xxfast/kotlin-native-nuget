- [018](018-type-alias-mapping.md) ‚Äî Transparent expansion to underlying type; aliases erased (matches all interop targets)
- [019](019-suspend-function-mapping.md) ‚Äî Callback-based bridge with TaskCompletionSource; suspend ‚Üí Task/async; coroutine launched on Dispatchers.Default
- [020](020-suspend-lambda-mapping.md) ‚Äî `KotlinSuspendFunc<R>` / `KotlinSuspendAction` with `InvokeAsync()` for suspend lambda properties
- [021](021-structured-concurrency.md) ‚Äî Scope-per-class with `SupervisorJob`; `Dispose()` cancels in-flight coroutines; `ObjectDisposedException` guard
- [022](022-cancellation-token-support.md) ‚Äî `CancellationToken` parameter on async methods; Job StableRef returned from export; `nuget_job_cancel` shared export
- [023](023-exception-propagation.md) ‚Äî `KotlinException.KotlinType` property; `Pair<String,String>` error bridge; `nuget_error_type`/`nuget_error_message` exports
- [024](024-sync-exception-propagation.md) ‚Äî Error out-parameter for sync functions; wrap-all (no `@Throws` required); mirrors ObjC `NSError**` ABI
- [025](025-async-disposable.md) ‚Äî `IAsyncDisposable` + `DisposeAsync()` for graceful drain; `nuget_scope_drain` export joins all child jobs before releasing handles
- [026](026-flow-mapping.md) ‚Äî `Flow<T>` ‚Üí `KotlinFlow<T>` (`IAsyncEnumerable<T>`); Channel-buffered push‚Üípull bridge; three-callback ABI (`onNext`, `onComplete`, `onError`)
- [027](027-stacktrace-propagation.md) ‚Äî `KotlinException.KotlinStackTrace` property; `Triple<String,String,String>` error bridge; `nuget_error_stacktrace` export; `ToString()` override appends Kotlin trace
- [028](028-exception-cause-chain.md) ‚Äî `e.cause` ‚Üí `KotlinException.InnerException`; recursive `NugetError` data class error bridge (replaces `Triple`); `nuget_error_cause_*` exports; each cause a full `KotlinException`
- [029](029-exception-type-mapping.md) ‚Äî `IKotlinException` interface; 8 Kotlin stdlib exceptions mapped to .NET analogs (e.g. `IllegalArgumentException` ‚Üí `KotlinArgumentException : ArgumentException`); `KotlinException` fallback for unmapped types; `BuildMapped` switch in generated helper
- [030](030-property-exception-propagation.md) ‚Äî ADR-024 error-out-parameter extended to property getter/setter exports; both callable forms of nullable two-call pattern wrapped independently
- [031](031-constructor-exception-propagation.md) ‚Äî Constructor export returns `COpaquePointer?` (null on error) + `errorOut`; C# constructor checks error before assigning `_handle`; data class `copy()` receives same treatment
- [032](032-generic-constructor-exception-propagation.md) ‚Äî ADR-031 error-out applied to all 14 generic `create_*`/`create_object` exports; C# error check centralized in `NugetMarshal.CreateBox<T>` (unconstrained) + per-constructor for constrained `Create_object`
- [033](033-value-class-constructor-exception-propagation.md) ‚Äî Value-class ctor exceptions via private `CreateChecked` helper called from `: this(...)` (keeps `new Email(...)`, no static-factory redesign); primitive underlying uses ADR-024 dummy-return pattern
- [034](034-secondary-constructor-exceptions.md) ‚Äî Export all constructors via `getConstructors()`; primary keeps `_create`, secondaries get `_create_N`; ADR-031 error-out per constructor; fail-fast KSP error on C# signature collision
- [035](035-value-class-primary-constructor-validation.md) ‚Äî Value-class primary `init` enforced by switching to a hand-written `record struct` where every ctor has a validating body (also removes ADR-033's `: this(...)` delegation); get-only property prevents `with`/initializer bypass; reference-underlying deferred
- [036](036-reverse-interop-mechanism.md) ‚Äî C#‚ÜíKotlin reverse interop via delegate + `GCHandle.Alloc` + `Marshal.GetFunctionPointerForDelegate`; consistent with existing Flow/async callback pattern; `[UnmanagedCallersOnly]` and vtable interface bridging deferred
- [037](037-stored-callbacks.md) ‚Äî Stored callbacks (event handlers/observers) via `IDisposable` subscription handle; `add*`/`remove*` pair detection; bridge-lambda registry on Kotlin side; no new Kotlin‚ÜíC# mechanism needed
- [038](038-aot-compilation.md) ‚Äî NativeAOT compatibility; generics reflection path (`GetField`/`Activator`) is the concrete trimmer blocker; `CSharpProfile` makes the dialect (`DllImport`‚Üí`LibraryImport`, generic dispatch) a generation-time choice; deferred
- [039](039-interface-bridging.md) ‚Äî C# implementing Kotlin interfaces via N flat function pointers per method; `add*/remove*` pair detection (same as ADR-037); Kotlin bridge generates anonymous `object : Interface`; same `IListener` for both directions; `IDisposable` subscription
- [040](040-interface-return-type-mapping.md) ‚Äî Kotlin returning interface-typed values to C#; concrete `sealed class Foo : IFoo` per interface; Kotlin interface-dispatch exports (`asStableRef<Foo>().get()`); `IPet?` return type; fixes current broken `new Pet(handle)` compile error
- [041](041-kotlin-to-csharp-call-mechanism.md) ‚Äî Phase 8 Kotlin ‚Üí managed C# calls via `[UnmanagedCallersOnly]` thunks registered at startup from `[ModuleInitializer]`; Kotlin stubs `requireNotNull`-guard the pointer table and fail fast if unregistered; standalone Kotlin hosts requiring `hostfxr` CLR hosting explicitly out of scope for v1
- [042](042-assembly-metadata-extraction.md) ‚Äî Phase 8 metadata extraction: `dotnet restore` (synthetic `.csproj` ‚Üí `project.assets.json`) for resolution + bundled dotnet tool using `System.Reflection.Metadata.MetadataReader` to read the ECMA-335 public API without loading the assembly; pure-JVM both halves deferred as Future Improvement
- [043](043-bridgeable-subset-boundary.md) ‚Äî Phase 8 C-ABI-bridgeable subset: overload sets, `ref struct`/`Span<T>`, open generics, `dynamic`, default interface methods structurally excluded; v1 auto-mappable subset only + per-member fail-fast diagnostics naming each skipped member; Kotlin subclassing a C# class out of scope, implementing a C# interface in scope
- [044](044-nuget-dependency-dsl.md) ‚Äî Phase 8 NuGet dependency DSL: `dependency()` + `bind {}` opt-in inside `nuget { dependencies { } }`; absent `bind {}` = resolve only; `packageName`/`include`/`exclude`/`alias` in `bind`; downstream data contract for `nugetGen`/`nugetExtractApi` tasks
- [045](045-nuget-resolution-pipeline.md) ‚Äî Phase 8 resolution pipeline: two-task split `nugetGen` (pure csproj generation, cacheable) + `nugetRestore` (`dotnet restore`, `@DisableCachingByDefault`); `net8.0` TFM floor; conditional `<RestoreSources>`; `obj/project.assets.json` as inter-task manifest; `nugetImport` umbrella task; config-cache via `ExecOperations`
- [046](046-reverse-ir-model-and-json-contract.md) ‚Äî Phase 8 Reverse IR: `RirFile`/`RirAssembly`/`RirNamespace`/`RirType`/`RirMethod`/`RirProperty`/`RirTypeRef` sealed data classes in `nuget/` module + `kotlinx.serialization` JSON with `"kind"` discriminator; `parseReverseIr()` + `deriveDllPaths()` pure functions; `NugetExtractApiTask` inputs/outputs + plugin wiring; `nuget-metadata-reader/` C# `MetadataReader` project; five-slice TDD breakdown
- [047](047-per-package-namespace-filters-at-reader-cli.md) ‚Äî Phase 8 per-package namespace filters: inline stateful `--include`/`--exclude` flags adjacent to each `--package` triple (vs JSON manifest); fixes flattening bug where Package A's filters were applied to Package B's assemblies; `metadataReaderCommand` signature changes from `List<String>` to `Map<String, List<String>>` for includes/excludes; `CliArgs.Parse` in Program.cs becomes stateful
